Module 10

Integration & APIs

~15 min read  ·  8-question quiz

The Enterprise Integration Problem

A typical enterprise uses dozens — sometimes hundreds — of software systems. The CRM tracks customers. The ERP manages finance and inventory. The marketing platform runs campaigns. The e-commerce site processes orders. The support system handles tickets. Each of these systems was probably purchased at different times, from different vendors, and built on different technologies.

The problem: data trapped in one system isn't available to the others. A closed deal in the CRM doesn't automatically create an invoice in the ERP. A new customer in the billing system doesn't automatically create an account in the support portal. Without integration, employees manually copy data between systems — which is slow, error-prone, and expensive.

Integration is the connective tissue of the enterprise Individual systems are valuable. Connected systems are transformative. When the CRM, ERP, support platform, and analytics stack share data in real time, the business operates with a level of visibility and efficiency that disconnected systems can't match.

What Is an API?

An API (Application Programming Interface) is a defined way for two software systems to communicate with each other. Think of it as a waiter in a restaurant: you (one system) tell the waiter (the API) what you want, the waiter goes to the kitchen (another system), and comes back with the result.

Modern software is built with APIs in mind. Almost every enterprise SaaS product exposes an API so other tools can send it data or retrieve data from it. This is what makes integration possible.

REST APIs

The most common type of API today. REST (Representational State Transfer) APIs use standard web protocols (HTTP) and return data in JSON format. When you hear someone say "our product has an open API" or "we integrate with Salesforce via API," they almost always mean a REST API.

REST APIs use standard actions called HTTP methods:

  • GET — retrieve data
  • POST — create new data
  • PUT / PATCH — update existing data
  • DELETE — remove data

Webhooks

A webhook is a way for one system to notify another in real time when something happens. Rather than constantly checking ("did anything change?"), the source system sends a message the moment an event occurs.

Example: when a deal is marked Closed Won in Salesforce, a webhook fires instantly and sends the deal data to the ERP — triggering invoice creation automatically. Webhooks are how modern systems achieve real-time data flow without constant polling.

Integration Approaches

Point-to-Point Integration

Directly connecting two systems — System A talks directly to System B. Simple for small numbers of integrations, but as the number of systems grows, this creates a "spaghetti" of connections that becomes impossible to maintain.

iPaaS (Integration Platform as a Service)

A cloud platform that acts as a central hub for integrations. Instead of every system talking directly to every other system, they all connect to the iPaaS, which routes data between them. Much easier to manage at scale.

Major enterprise iPaaS vendors: MuleSoft (Salesforce), Dell Boomi, Workato, Informatica.

Simpler/low-code alternatives popular with mid-market companies: Zapier, Make (formerly Integromat), Tray.io.

Middleware / ESB (Enterprise Service Bus)

An older, on-premise approach to integration. ESBs were common before cloud iPaaS solutions existed. Many large enterprises still run on ESB infrastructure and are migrating to cloud iPaaS.

EDI (Electronic Data Interchange)

EDI is a standardized format for exchanging business documents between companies — purchase orders, invoices, shipping notices. It's old technology (predating the internet) but still extremely common in industries like retail, logistics, and manufacturing where large companies require their suppliers to communicate in EDI format.

If a major retailer wants to source products from a small manufacturer, they may require the manufacturer to send purchase order confirmations and invoices via EDI. This creates significant overhead for smaller suppliers.

Why Integration Matters for Consultants

Integration is one of the most common — and most underestimated — challenges in enterprise software implementations. A few things to remember:

  • Every implementation involves integration work. A new CRM must connect to the ERP, the marketing platform, the data warehouse, and the support system. Integration can easily represent 30–50% of total implementation cost.
  • Bad integration creates data quality problems. If the CRM and ERP have different customer records because the sync is broken, downstream decisions based on that data are unreliable.
  • APIs have rate limits and versioning. API connections can break when a vendor releases a new API version. This is a real maintenance cost that clients often don't anticipate.
  • "We'll just integrate it" is a red flag. When a client waves off integration concerns with "we'll figure it out," that's often where projects get into trouble.
The last mile of value Even the best individual software products can fail to deliver ROI if they're not integrated with the rest of the stack. Helping clients plan and execute integrations well is often where consultants add the most lasting value.
